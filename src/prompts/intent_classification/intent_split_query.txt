Split the user query into distinct, independent sub-questions for intent classification.

Output ONLY valid JSON. No extra text, no explanation, no markdown.

Output format:
{"intents": ["sub-question 1", "sub-question 2"]}

Rules:
1. If the query contains only one intent, return a list with one item.
2. Do NOT merge separate questions into one.
3. Preserve all entities exactly: ASINs, Order IDs, dates, numbers, time ranges.
4. Do NOT split date strings with commas (e.g. "January 1, 2025" stays whole).
5. Do not add or remove information.
6. Do NOT answer the question.

Examples:
Input: "what is FBA get order 123 which table stores fees"
Output: {"intents": ["what is FBA", "get order status for 123", "which table stores referral fee data"]}

Input: "show me sales last month"
Output: {"intents": ["show me sales last month"]}

Input: "what is FBA vs FBM get order 112-123 show storage trend"
Output: {"intents": ["what is the difference between FBA and FBM", "get order status for 112-123", "show me month over month trend of storage fees"]}
